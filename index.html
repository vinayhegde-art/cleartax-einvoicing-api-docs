Overview

Global e-Invoicing — Unified API

A single API integration to generate, validate, and submit compliant e-invoices across multiple countries. ClearTax validates your payload and routes it to the appropriate government platform or Peppol network based on the country specified in the request header.

One schema, multiple countries. The Unified API uses a single JSON payload structure inspired by UBL and PINT. Country-specific fields are added via an extensions object, keeping your core integration unchanged as you expand to new markets.

How to Use the API

Onboard on ClearTax — Create a workspace on the demo environment for testing and a production workspace for live invoices.
Add countries — Enable the countries you operate in from your workspace settings.
Configure business entities — Add entity identifiers (VAT / TIN / TRN) and branches under Business Settings.
Generate an API client and Auth Token — Each workspace has its own auth token. Belgium, Malaysia, and UAE require separate tokens.
Build your payload — Start with common fields, then add country-specific extensions for your target country.
Send and track — Submit via the Send API, track processing via Status Check, and optionally trigger PDF generation or buyer email.

Authentication

All requests must include a valid auth token. The business identifier header is conditional on the country.

API Reference

Send e-Invoice

Submit one or more invoice documents for validation and routing. ClearTax validates the payload, transforms it to the country-specific format, and submits to the government platform or Peppol network.

POST/einvoice/v1/documents/generate
Supports up to 10 documents per request. DocumentData must be a Base64-encoded JSON string of the eInvoice object.

Request Headers

See the Authentication section for the full header reference.

Query ParameterTypeDescription
einvoice-typeEnumDefault: SALES. Currently only SALES is supported.

Request Body

FieldTypeRequiredDescription
DocumentsArrayMandatoryList of invoice documents. Min: 1, Max: 10.
Documents[].DocumentDataStringMandatoryBase64-encoded JSON of the eInvoice object.
Documents[].DocumentFormatEnumMandatoryFormat of the document data. Only JSON is supported.
Documents[].CustomFieldsObjectOptionalKey-value pairs for custom fields in PDF templates or reports.

Response Status Codes

200Success. Invoice accepted and processed.
400Bad Request. Invalid payload or missing fields.
401Unauthorized. Auth token missing or invalid.
403Forbidden. User lacks permission for this entity.
409Conflict. Duplicate document ID.
429Rate Limit Exceeded.
500Internal Server Error.
503Service Unavailable.

Status Check

Retrieve the current processing status of a previously submitted document using the DocumentId returned from the Send e-Invoice API.

GET/einvoice/v1/documents/{document_id}/status

Request Headers

Same authentication headers as the Send e-Invoice endpoint. See Authentication.

Path ParameterTypeRequiredDescription
document_idStringMandatoryThe DocumentId returned from the Send e-Invoice response.

Response Schema

FieldTypeDescription
SuccessBooleanWhether the status check request was processed successfully.
DocumentIdStringThe ClearTax document ID.
StatusEnumPending, Failed, Submitted to exchange network, Delivered to buyer network
ErrorDetailsArrayOptional. List of error objects if processing failed.

Sample Response

JSONHTTP 200 — Success
{
  "Success": true,
  "DocumentId": "686d022e418c8b4e000a1feb",
  "Status": "Delivered to buyer network",
  "ErrorDetails": null
}
JSONHTTP 404 — Not Found
{
  "ErrorDetails": [{
    "ErrorCode": "300102",
    "ErrorMessage": "No data found for the selected Document ID",
    "ErrorSource": "CLEARTAX",
    "Path": null
  }]

UAE AE

The UAE adopts e-invoicing through the Peppol network using the PINT AE specification. VAT is governed by the Federal Tax Authority (FTA). ClearTax routes UAE invoices through the MEA (Middle East and Africa) regional infrastructure.

AED reporting required. UAE invoices must include total and line-level amounts in AED in the PINT AE extension fields, even when the invoice is denominated in a foreign currency.
